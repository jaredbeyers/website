Packet Sniffing & Traffic Analysis

Network+ Project: Capturing and analyzing network traffic in a virtual lab environment

Project Overview

This project demonstrates practical application of Network+ skills through packet capturing and analysis in a virtual lab environment. The goal is to capture and analyze network traffic between virtual machines to understand protocols, IP addressing, ports, and detect anomalies, simulating real-world network monitoring tasks.

Tools & Technologies

VirtualBox Kali Linux Wireshark Windows/Linux VMs Network Analysis
  • Virtualization: VirtualBox for creating isolated network environments
  • Operating Systems: Kali Linux (Wireshark VM), Windows/Linux VMs as traffic generators
  • Network Tool: Wireshark for packet capture and analysis
  • Other Tools: Ping, Browser, Optional: Local Web Server

Network Setup

Topology

+---------+          +---------+
| VM1     |          | VM2     |
| Kali    |----------| Windows |
| (Wireshark)        |/Linux  |
+---------+          +---------+
  • Network Type: Internal Network (VirtualBox) - "LabNet"
  • IP Addresses:
    • VM1 (Wireshark): 192.168.56.10
    • VM2: 192.168.56.11
    • VM3 (optional): 192.168.56.12
  • Setup Steps:
    • Create VMs and assign them to the same internal network (LabNet)
    • Verify connectivity via ping: ping 192.168.56.11

Packet Capture Procedure

  1. Open Wireshark on VM1
  2. Select the network interface connected to LabNet
  3. Start capturing traffic
  4. Generate network activity on VM2/VM3:
    • Ping other VMs: ping 192.168.56.10
    • Browse websites (if internet accessible)
    • Optional: Run a small HTTP server on VM2 and connect from VM3
  5. Apply filters in Wireshark to focus on traffic of interest:
    ICMP: icmp
    HTTP: http
    Specific IP: ip.addr == 192.168.56.11
  6. Stop capture after sufficient traffic is observed
  7. Save capture as .pcap file for analysis

Key Observations

  • ICMP packets demonstrate connectivity between VMs
  • TCP packets show communication over HTTP (port 80)
  • No unexpected traffic detected
  • Source and destination IPs align with the lab topology

Identifying Anomalies

For practice, anomalies can be simulated to test detection capabilities:

  • Repeated ping floods: High ICMP traffic indicating potential DoS attempts
  • Unrecognized ports/protocols: Unknown services or potential security risks
  • DNS queries: Observation of DNS queries during browsing activities

Lessons Learned

  • Understanding how protocols operate (ICMP, TCP, UDP, HTTP)
  • How source/destination IPs and ports are used to route traffic
  • Using Wireshark filters for focused analysis
  • Basics of detecting suspicious network activity
  • Network setup and IP addressing in a virtual environment
  • Practical application of Network+ certification concepts

Conclusion

This lab demonstrates practical application of Network+ skills, including network setup and IP addressing in a virtual environment, packet capturing and filtering with Wireshark, understanding protocols and ports, and identifying potential anomalies in network traffic. The project provides hands-on experience with network monitoring tools and techniques essential for network administration and security analysis.
